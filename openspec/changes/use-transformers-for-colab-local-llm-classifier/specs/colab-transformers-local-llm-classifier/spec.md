## ADDED Requirements

### Requirement: Notebook uses Transformers for local classifier inference
The Colab local LLM classifier notebook SHALL run classifier inference by loading the configured Hugging Face model and tokenizer through Transformers in the notebook process, without starting a vLLM server or using an OpenAI-compatible client.

#### Scenario: Model loads locally
- **WHEN** the notebook setup cells are executed on a Colab GPU runtime
- **THEN** the configured Hugging Face model and tokenizer are loaded with Transformers and the notebook reports that the local model is ready

#### Scenario: vLLM server is not used
- **WHEN** classifier inference is run from the notebook
- **THEN** the notebook does not start `vllm.entrypoints.openai.api_server`, does not poll `/v1/models`, and does not instantiate an OpenAI client

### Requirement: Notebook preserves classifier prediction outputs
The notebook SHALL continue writing classifier-only checkpoint and final parquet files with the existing prediction columns and compatible normalized classifier values.

#### Scenario: Successful classification row
- **WHEN** a row is classified and the model response parses as classifier JSON
- **THEN** the output row includes the existing `llm_*` and `clf_*` prediction columns, `llm_stages_run` equals `1`, and the classifier label, category, confidence, evidence, raw response text, parse status, provider name, model name, and token logprobs are populated

#### Scenario: Parse failure row
- **WHEN** a generated classifier response cannot be parsed as valid classifier JSON
- **THEN** the notebook writes a normalized parse-failure prediction row using the existing fallback semantics

### Requirement: Notebook records Transformers provider metadata
The notebook SHALL identify predictions generated by the migrated local inference path with a Transformers-local provider name while retaining the configured model name in output metadata.

#### Scenario: Provider metadata is written
- **WHEN** the notebook writes prediction rows after the migration
- **THEN** `llm_provider_name` and `clf_provider_name` indicate local Transformers inference and `llm_model_name` and `clf_model_name` contain the configured model alias

### Requirement: Notebook derives generated-token logprobs
The notebook SHALL derive a compact generated-token logprob summary from Transformers generation scores and store it in the existing token logprob output field.

#### Scenario: Logprobs are available
- **WHEN** Transformers generation returns per-step scores for generated tokens
- **THEN** the notebook records selected generated tokens with their logprob values in `clf_token_logprobs`

#### Scenario: Logprobs are unavailable
- **WHEN** token logprobs cannot be derived for a generated response
- **THEN** the notebook still completes classification and stores a null or empty token logprob value without changing the prediction schema

### Requirement: Notebook handles local inference failures without server health checks
The notebook SHALL fail setup errors directly and handle row-level inference errors without vLLM server health probes.

#### Scenario: Model setup fails
- **WHEN** Transformers model or tokenizer loading fails
- **THEN** the setup cell raises an actionable error in the notebook output without referencing a vLLM server log

#### Scenario: Row inference fails
- **WHEN** classification for an individual row raises an exception after model setup has succeeded
- **THEN** the notebook records or reports the row failure using the existing parse-failure row path and does not perform an HTTP health check
